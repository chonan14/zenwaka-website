# 全国若手議員の会 ホームページリプレイス 要件定義書

作成日: 2026-04-23
対象ドメイン: https://www.zenwaka.net

---

## 1. プロジェクト概要

全国若手議員の会の公式ホームページを全面リプレイスする。現行WordPressサイトを廃止し、素のHTML/CSS/JSによる静的サイトとして再構築する。「全体像を固めてから構築に着手」する方針。

### 目的

- **対外的な周知（名刺代わり）** — マスコミ・新人若手議員に対し、団体の価値と活動実態を伝える
- 情報量不足という現行サイトの課題解消
- 入会導線・問合せ導線の確立

### 成功指標（目安）

- 公開後、新人議員からの入会問合せ発生
- マスコミの取材窓口として機能
- 活動の3本柱（キャラバン／政策提言／研修）が外部から見える状態

| 項目 | 内容 |

## 4. デザイン方針

### コンセプト

**無印良品寄りの日本語ミニマル**。Apple Japanのような1画面1メッセージの考え方を取り入れつつ、日本語の情報量に合わせて無印のバランス感を採用。

### ビジュアル要件

- ロゴ: 既存（支給あり）
- メインカラー: **オレンジ**（アクセント1色として使用）
- ベース: 白・黒・淡いグレーの無彩色
- タイポグラフィ: Noto Sans JP 等の現代的ゴシック、広い行間
- 写真: 報道調のモノクロまたは低彩度に統一し、政治色を抑える
- トーン: シック・信頼感・超党派の中立性

### オレンジの使いどころ

CTAボタン／現在地インジケータ／強調したい数字などに限定。面積を絞ることでシックさを保つ。

### 参考サイト

- https://www.apple.com/jp/ — 余白と1画面1メッセージの考え方
- https://www.muji.com/jp/ja/ — 日本語主体のミニマル
- https://florence.or.jp — 白×ネイビー＋オレンジアクセント（配色参考）
- https://www.katariba.or.jp — 非営利の誠実さ×ミニマル

## 6. 必須コンテンツ要素（オーナー指定）

トップ〜各所に以下の要素を必ず含めること：

- 会の目的
- 活動の柱（キャラバン／政策提言／議員研修）
- 歴史（過去の人たちの名前を掲載）
- 会長挨拶
- 事務局長挨拶
- 役員紹介（部会長まで写真あり、他は文字のみ）
- 研修報告
- 問合せフォーム
- 入会フォーム
- 名簿のURL（外部リンク）

## 7. 機能要件

### 7.1 入会フォーム

**項目**
- 氏名
- 郵便番号
- 住所
- 携帯電話番号
- メールアドレス
- 自治体名
- 当選年

**通知先**: 要確認

### 7.2 問合せフォーム

**通知先**: 要確認（マスコミ窓口と一般問合せを分ける設計を推奨）

### 7.3 翻訳

- ブラウザ翻訳 or Google翻訳ウィジェット（仕様要確認）
- 多言語ページは作らず、翻訳機能のみ

### 7.4 アクセス解析

- GA4 導入

### 7.5 CTA配置

- ヘッダー右上に常時「入会」ボタン
- フッター上に再掲
- 各ページ末尾にセカンダリCTA

## 8. 技術要件

| 項目 | 内容 |
|---|---|
| 構成 | 素のHTML / CSS / JS（ビルド不要） |
| ホスティング | Cloudflare Pages または Vercel（推奨: Cloudflare Pages） |
| ドメイン | zenwaka.net（DNS管理者: オーナー） |
| リダイレクト | 旧URL→新URLのマッピング不要 |
| CMS | なし |
| SSL | ホスティング側で自動付与 |
| パフォーマンス目標 | LCP 0.5〜1.0秒以内 |
| アクセシビリティ | WCAG AAを目安（要合意） |

## 9. 運用・更新

- 更新担当: オーナー本人
- 更新方式: HTMLを直接編集（Git運用の可否は要確認）
- CMS: 不要
- 更新頻度: お知らせ／研修報告の随時更新

---

## 10. 差別化ポイント

1. **超党派・自治体現場発を数字で可視化** — 会員数・参加自治体数・開催研修数をトップに大きく配置
2. **9ブロック×4分科会を日本地図でビジュアル化** — Code for Japanのブリゲード表示の和製版
3. **政策提言のオープン化** — 過去提言のPDFアーカイブを検索可能にし、**マスコミがネタ元として使える資料室**にする（「名刺代わり」最大の武器）
4. **マスコミ専用窓口**を独立CTA化
5. 静的HTML × Cloudflare Pagesの軽量性による**高速表示**
